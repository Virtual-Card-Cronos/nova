This Product Requirements Document (PRD) outlines the NovaAgent Hybrid Platform, a dual-mode commerce system. It serves two distinct customer types: Human Users (Manual Web3 Checkout) and AI Agents (Autonomous x402 Protocol).PRD: NovaAgent Hybrid — Multi-Modal Agentic Commerce1. Vision & GoalsNovaAgent aims to be the premier "Agent-Ready" retail bridge on the Cronos EVM. By combining a polished human storefront with an x402-compliant API, we enable a future where users can shop manually or delegate a shopping agent to find, negotiate, and purchase gift cards autonomously.2. Core Features & Capabilities3.1 Dual-Channel Frontend (Next.js)Manual Storefront: A grid-based UI displaying the Tillo Catalog with "Buy Now" buttons triggering traditional wallet signatures.Agent Console: A real-time terminal where users monitor their agents' activities, spend, and "thought processes" via the Crypto.com AI Agent SDK.3.2 Hybrid Payment EngineTraditional Web3 Path: Standard transaction flows for humans using thirdweb Connect and Crypto.com Pay.x402 Agentic Path: An autonomous handshake where the server issues an HTTP 402 Payment Required challenge, and the agent satisfies it via an EIP-712 signed authorization.3.3 The Aegis Policy Layer (Smart Contract)An on-chain Guardrail Contract on Cronos EVM that governs agent behavior.Policies: Daily spending caps, whitelisted brands (e.g., "Only Amazon/Steam"), and max transaction limits to prevent "rogue agent" scenarios.3.4 Unified Fulfillment Bridge (Tillo v2)A server-side "Relay" that listens for successful payments from either channel.Once payment is verified (via x402 Facilitator or On-chain Event), the bridge calls api.tillo.tech/v2/digital/issue.4. Technical Architecture4.1 The StackFrontend: Next.js 15, Tailwind, thirdweb SDK v5.Agent Logic: Crypto.com AI Agent SDK + Market Data MCP Server.Settlement: Cronos x402 Facilitator (for gasless agentic payments).Database: Neon (Postgres) for order tracking and Tillo brand caching.Notifications: Resend for instant digital code delivery.5. Detailed User Journeys5.1 Manual Path (The "Human" Flow)User connects wallet $\rightarrow$ Selects Brand $\rightarrow$ Clicks "Buy".Next.js triggers a standard USDC transfer on Cronos.Server verifies TX hash $\rightarrow$ Calls Tillo API $\rightarrow$ Shows code on screen.5.2 Agentic Path (The "X402" Flow)User delegates task: "Find a $10 UK Steam card and buy it."NovaAgent Brain queries Tillo catalog (cached in Neon).Agent calls /api/v1/agent/purchase.Server responds with 402 Payment Required.Agent signs authorization $\rightarrow$ Retries with X-PAYMENT header.Facilitator settles on Cronos $\rightarrow$ Server calls Tillo $\rightarrow$ Agent notifies user.6.
